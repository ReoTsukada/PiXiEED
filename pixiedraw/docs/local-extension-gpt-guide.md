# PiXiEEDraw Local Extension GPT Guide

PiXiEEDraw の `拡張ファイルを作る` で出る `.js` は、そのまま GPT に渡して編集できます。  
この説明書は、GPT に「何を守って編集するか」を伝えるための前提です。

## 渡すもの

- PiXiEEDraw が作成した `.js` テンプレファイル
- この説明書

## 実際に GPT に渡す時の形

おすすめは次の 2 点セットです。

- 生成した `.js` テンプレ本体
- この説明書

理由:

- `.js` の先頭コメントに、実行環境と禁止事項を埋め込んである
- この説明書には、より長い前提と依頼文の型がある
- 片方だけより、GPT が誤読しにくい

## この拡張の実行環境

- 拡張は `この端末だけ` で動く
- 拡張は `sandbox iframe` の中で実行される
- 拡張パネルの shell は最初から用意されている
- UI は `api.ui.getRoot()` で取得できる専用 root の中に作る
- `canvas` を好きな数だけ作れる
- ローカル保存は `api.storage.get/set/remove()` を使える
- 外部通信はできない
- 親ページ DOM には触れない
- 共有状態や他人の画面には触れない
- 既存ツール欄やメインキャンバスには触れない
- 専用パネル内だけで完結する

## 使える API

- `api.on(name, handler)`
  イベント購読
- `api.toast(message, level)`
  拡張パネル内のステータス表示
- `api.ui.getRoot() / clear() / mount(node) / append(node)`
  既存 shell の中へ UI を出す
- `api.ui.setTitle(text) / setSubtitle(text) / setStatus(text, kind)`
  shell 見出しと状態表示を変える
- `api.ui.addStyles(css)`
  拡張専用 CSS を追加する
- `api.ui.el(tag, props, children)`
  要素作成の簡易 helper
- `api.getContext()`
  現在の簡易コンテキスト取得
- `api.storage.get/set/remove()`
  端末内だけの保存

## よく使う event

- `init`
- `context`
- `interval`

## GPT に守らせる条件

以下をそのまま渡してください。

```text
このファイルは PiXiEEDraw のローカル拡張です。

必ず守ること:
- 単一の .js ファイルのまま編集する
- 外部ライブラリを使わない
- 外部通信しない
- 親ページの DOM は触らない
- 既存の拡張パネル shell は api.ui から使う
- PiXiEEDraw 本体の共有データは直接変更しない
- 既存のツール欄やメインキャンバスへ機能追加しない
- 既存のコード構造とコメントをできるだけ保つ
- 壊してよいのは依頼した機能の範囲だけ
- 完成コードだけを返す
```

## 今のテンプレの構造

テンプレは大きく 8 セクションです。

- `1) Config`
  サイズ、色、方向セットなどの定数
- `2) State`
  現在の編集方向、4方向の元データ、hover、drag 状態
- `3) Helpers`
  配列、signed field 補完、保存、基本ユーティリティ
- `4) Panel shell + DOM`
  api.ui の shell に中身を組み立てる
- `5) Direction set`
  4方向から 8方向を組み立てる補完ルール
- `6) Rendering`
  編集キャンバス、8方向プレビュー、統計表示の更新
- `7) Editor input`
  クリック、ドラッグ、消去などの入力処理
- `8) Init`
  起動時処理

## GPT に頼みやすい変更例

- UI をもっと大きくしたい
  - `Config` のサイズ値と `DOM` / `Rendering` を調整してもらう
- 4方向ではなく 3方向だけ編集したい
  - `MASTER_DIRECTIONS` と `buildDirectionSet()` を調整してもらう
- 補完方法をもっと強くしたい
  - `buildDirectionSet()` の派生ルールを変えてもらう
- 24x24 ではなく 32x32 にしたい
  - `GRID_SIZE` と canvas サイズを変更してもらう
- サンプル形状を別の形にしたい
  - `applySampleSprite()` だけ変えてもらう
- 方向ごとのタグやメモを持たせたい
  - `State` と `Rendering` を拡張してもらう
- プレビューを 2 列ではなく 1列や 4x2 にしたい
  - `preview-grid` の CSS と preview card 周りを変えてもらう

## GPT に渡す依頼文の型

```text
添付した PiXiEEDraw ローカル拡張テンプレを編集してください。
添付した guide の制約を守ってください。

変更したい内容:
- ここにやりたいことを書く

追加条件:
- UI は今より崩さない
- 既存の保存処理は残す
- 単一ファイルのまま返す
- 完成コードのみ返す
```

## 依頼文の具体例

### 例1: 8方向プレビューを 4x2 固定にする

```text
添付した PiXiEEDraw ローカル拡張テンプレを編集してください。
guide の制約を守ってください。

変更したい内容:
- 8方向プレビューを常に 4列 x 2行で表示する
- それぞれの方向名をもう少し大きくする
- 派生方向には「AUTO」と明記する

追加条件:
- 既存の 4方向編集機能は壊さない
- 保存処理はそのまま残す
- 完成コードのみ返す
```

### 例2: 斜め方向を 2 枚だけで回したい

```text
添付した PiXiEEDraw ローカル拡張テンプレを編集してください。
guide の制約を守ってください。

変更したい内容:
- 編集方向を N / E / S の 3方向に減らす
- NE / SE / SW / NW は残った方向から補完する
- buildDirectionSet() の補完ルールを新しい仕様に合わせて更新する

追加条件:
- UI 全体のトーンは今のまま
- 単一ファイルのまま返す
- 完成コードのみ返す
```

## GPT に伝えると事故が減ること

- どの関数だけ触ってほしいか
- 見た目も変えたいか、機能だけ変えたいか
- 既存の保存データを壊していいか
- 既存のレイアウトを維持したいか
- 速度重視か、見た目重視か

## 向いている改造

- 独自 UI
- 独自 canvas
- ローカル保存
- 補助プレビュー
- 音や簡単なアニメーション
- 作業補助パネル

## 安全に追加しやすいもの

- 第2キャンバスや第3キャンバス
- プレビュー専用表示
- 作業メモ
- 補助UI
- ローカルだけの音
- ローカルだけの保存付き設定

## 向いていない改造

- 共有データの直接編集
- 外部API連携
- 他人の端末への反映
- 親ページそのものの改造
