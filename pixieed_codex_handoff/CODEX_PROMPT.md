# PiXiEED トップページ改修プロンプト

## 目的
既存の PiXiEED トップページを、添付の `reference-layout.jpg` に近い情報設計・レイアウトへ改修してください。

参考画像の見た目をそのまま複製するのではなく、現在の PiXiEED の画像・アイコン・配色・世界観・既存機能を維持したまま、主に次を改善してください。

- 情報の優先順位
- デスクトップの2カラム構成
- モバイルのアプリ風ホーム画面
- PiXiEEDrawへの導線
- PiXiEELENS、アニメーション、共有、素材、記事の整理
- 更新情報と初心者向け導線
- SEO向けテキストとアプリ風UIの両立

参考画像:
- `reference-layout.jpg`

## 最重要条件
1. 既存のアイコンや画像は大幅に変更しない。
2. 既存アセットを優先的に再利用する。
3. 新しい画像生成や別テイストへの差し替えは不要。
4. 主にHTML構造、CSSレイアウト、余白、カード階層、レスポンシブ設計を修正する。
5. ログイン、プロフィール、広告、ギャラリー、共有キャンバス、更新情報、ボトムナビなど既存機能を壊さない。
6. JavaScriptが参照するID、class、data属性は必要がない限り変更しない。
7. 既存URLを維持し、リンク切れを起こさない。
8. 全面書き換えではなく差分ベースで実装する。

## 最初に確認すること
作業前にリポジトリを調査し、次を報告してください。

- トップページで実際に使われているHTML/CSS/JS
- `index.html` が読み込むCSS
- 既存JSが参照するID・class・data属性
- 既存のモバイル下部ナビ構造
- 既存のカルーセル、ギャラリー、広告、プロフィール処理
- 参考画像との差
- 変更対象ファイル

想定対象:
- `index.html`
- `styles.css`
- `scripts.js`
- `site/landing.css`
- その他、トップページ専用CSS/JS

`index.html` では `site/landing.css` が使われている可能性があるため、どのCSSが実際の表示を制御しているか必ず確認してください。

## デスクトップ構成
ページ上部を参考画像に近い2カラム構成にしてください。

### 左側
1. ブランドヘッダー
2. 大型ヒーロー
3. 主要CTA
4. 機能タグ
5. ツールカード一覧

大型ヒーローで次を明確にしてください。

- 無料で使える
- ブラウザで使える
- ドット絵を描ける
- ドット絵を動かせる
- 画像をドット絵に変換できる
- 共有制作できる

主要CTA:
- `今すぐ描く / PiXiEEDraw`
- `画像をドット絵に変換 / PiXiEELENS`

PiXiEEDrawを最も目立つ主CTAにしてください。

### 右側
1. 最新情報
2. 初めての方向けステップ
3. 必要に応じて注目記事

最新情報は縦リスト形式にし、更新種別・日付・短いタイトルを確認しやすくしてください。

初心者向けステップ:
1. 描く
2. 動かす
3. 共有する
4. 保存・出力する

既存画像または既存アイコンを使ってください。

## ツールカード
優先順位:
1. PiXiEEDraw
2. PiXiEELENS
3. アニメーション
4. QRコードリーダー
5. その他

各カード:
- 既存アイコン
- ツール名
- 1行説明
- 既存スクリーンショットまたは既存画像
- 明確な「開く」ボタン

PiXiEEDrawは最も大きく、または最も目立つカードにしてください。

## モバイル方針
単なる縮小ではなく、ネイティブアプリのホーム画面に近い構成にしてください。

### 上部
- safe-area維持
- ブランド
- 通知または既存右上操作
- 必要ならメニュー
- 高さを取りすぎない

### ヒーロー
- 既存ヒーロー画像を利用
- 画面幅いっぱいに近いカード
- 角丸
- 見出しと短い説明
- CTAをヒーロー内または直下に配置
- 最初の1画面で何ができるか分かる

### 主要メニュー
参考画像のような2列または3列グリッドで、次をアプリ風カードとして表示してください。

- 描く
- 変換
- 動かす
- 共有
- 素材
- 記事

各カード:
- 大きい既存アイコン
- 短い動詞ラベル
- 小さい補足名
- 44px以上を目安にしたタップ領域
- 角丸
- 押下状態

### 更新情報
- カード内縦リスト
- 1項目1〜2行
- 日付・種別は小さく
- 「もっと見る」を設置
- 最初の画面を圧迫しない

### 下部ナビ
既存ボトムナビを維持しつつ、見た目の優先順位を以下に寄せてください。

- ホーム
- ツール
- 描く
- お知らせまたは素材
- マイページ

中央の「描く」を少し大きな主操作として強調してください。
既存機能・リンク構造と矛盾する場合は、機能を優先し、見た目だけ調整してください。

## デザイン要件
- 現在のダークテーマを維持
- 既存の青・紫系アクセントを基本にする
- 参考画像の情報密度を参考にする
- 過剰な虹色・ネオン化は避ける
- 既存画像が主役になるようにする
- 角丸、余白、カード境界を統一
- 見出し、説明、CTAの階層を明確化
- デスクトップは情報量を増やしてよい
- モバイルは操作優先
- hoverだけに依存しない
- `:focus-visible` を維持
- 横スクロールを起こさない
- iPhone safe-areaを考慮
- ドット絵画像の `image-rendering: pixelated` を必要に応じて維持

## SEO要件
アプリ風UIにしても検索向け本文を削除しすぎないでください。

title案:
`無料のドット絵作成サイト｜スマホ・PC対応 PiXiEED`

description案:
`ブラウザですぐにドット絵を作成できる無料ツール。PiXiEEDrawで描く、PiXiEELENSで画像をドット絵に変換する、アニメーションや共有制作にも対応。スマホ・PCで利用できます。`

H1は1つにしてください。

推奨H1:
`無料で使えるドット絵作成ツール`

本文には不自然にならない範囲で次を含めてください。

- ドット絵作成サイト
- ドット絵を描く
- ブラウザ
- 無料
- スマホ
- PC
- ドット絵アニメーション
- 画像をドット絵に変換
- 共有制作

キーワードの羅列は禁止です。

## URL・リンク
内部リンクは運用上の正規URLへ統一してください。

確認対象例:
- `/pixiedraw/index.html`
- `/pixiedraw/`
- `/projects/pixiedraw/`
- `/pixiee-lens/index.html`
- `/qr-maker/index.html`

リポジトリとサーバー設定から正規URLを確認し、トップページ内リンクを統一してください。

このタスクではサーバー側301リダイレクトを勝手に追加せず、必要なら別タスクとして報告してください。

## パフォーマンス
- 新しい大型ライブラリを追加しない
- 不要な外部依存を増やさない
- 既存画像を再利用
- above-the-fold画像を増やしすぎない
- lazy loadingを維持
- 画像領域にaspect-ratioを設定
- JSでレイアウトを生成しすぎずHTML/CSS中心
- 既存カルーセル処理を壊さない

## レスポンシブ確認
最低限、次の幅で確認してください。

- 360px
- 390px
- 430px
- 768px
- 1024px
- 1280px
- 1440px

目安:
- 0〜479px: スマートフォン
- 480〜767px: 大型スマートフォン
- 768〜1023px: タブレット
- 1024px以上: デスクトップ
- 1280px以上: 2カラムを十分に活用
